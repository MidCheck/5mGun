# 01 · 游戏设计大纲（GDD）

> 版本 v0.1 · 状态：待评审

---

## 1. 产品定位

| 维度 | 定义 |
|---|---|
| **品类** | 网页多人第一人称射击（Web Multiplayer FPS） |
| **目标用户** | 白领、上班族（25–40 岁），有 FPS 认知（玩过 CF/CS/吃鸡），但**时间碎片化** |
| **使用场景** | 午休、摸鱼、下班前、晚上睡前；和同事/好友开黑 |
| **单局时长** | **5–10 分钟**（硬约束，所有玩法围绕它设计） |
| **核心诉求** | 爽（释压）、快（即点即玩）、易（不用学）、聚（好友一起） |
| **差异点** | 免下载即玩 + 强爽点反馈 + 极短局时；不追求硬核拟真，追求"爽感密度" |

### 设计四支柱

1. **爽（Punch）** — 每一枪、每一次爆头、每一波击杀都有夸张的视听反馈。
2. **快（Instant）** — 从点链接到开打 ≤ 30 秒，无安装、无长加载、无复杂匹配。
3. **易（Pick-up）** — 默认武器开箱即用，移动端三指可玩，0 学习成本上手。
4. **聚（Social）** — 房间 + 邀请链接/口令，好友一键加入，天然开黑。

> 反向约束：**不做**长养成、不做开局选位、不做赛季天梯逼肝、不做超长地图。任何拖慢"开打"的设计都要砍。

## 2. 游戏模式

### 2.1 PvP（人打人）

**主推：团队歼灭 TDM（Team Deathmatch）**

- 队伍：**每房上限 20 人**（最多 10v10）；推荐快节奏档 4v4~5v5，大房档 8v8~10v10。
- 胜负：限时 **6 分钟** 或 **某队先达 N 杀**（N 按人数缩放，如小房 40 / 大房 80），先到为准。
- 复活：3 秒快速复活（保持节奏，避免死了干等）。
- 出生：双方固定出生区 + 反偷家保护（出生点 2 秒无敌）。
- **人机补位（Bot Fill）**：真人不足时自动用 AI 机器人填满两队，**1 个人也能立刻开一把**。机器人有难度档位（萌新/普通/老兵），房主可调；真人加入时机器人自动让位。详见 §2.4。

**子玩法：爆头模式（Headshot Only）** — 仅爆头造成伤害，强化"爽点 1"，作为趣味轮换。

**可选扩展（非首期）**：占点（Domination）、爆破（拆/保炸弹，CS 式，但缩短为单回合 90 秒）。

### 2.2 PvE（组队刷丧尸）

**血战上海滩式波次推进 + 守点**

- 队伍：**1–8 人**（单人也可开，难度按人数自动缩放，见 §2.4）。
- 结构：**5 大波 + 1 BOSS 波**，每波 60–90 秒，总时长约 8–10 分钟。
- 流程：清怪 → 短暂间歇（补给/升级）→ 下一波怪更强更多 → BOSS 收尾。
- 失败条件：全队同时倒地（多人支持队友扶起；**单人提供一次自救/限次复活**，避免一倒到底扫兴）。
- 奖励：局内即时掉金币（升枪/技能），局外结算经验与永久货币。

> PvE 是**成长爽点的主战场**：金币、升级、技能在这里即时生效，形成"打→变强→打更猛"的正反馈飞轮。

### 2.3 模式对比

| | PvP-TDM | PvE-丧尸 |
|---|---|---|
| 人数 | 1–20（不足由人机补位） | 1–8（难度按人数缩放） |
| 时长 | ≤6 min | 8–10 min |
| 核心爽点 | 对枪、爆头、连杀 | 割草、成长、爆BOSS |
| 命中判定 | 服务器权威 + 滞后补偿 | 服务器权威（怪由服务器模拟） |
| 难度来源 | 真人对抗（含 AI 机器人） | 丧尸数值与波次曲线（按人数缩放） |

### 2.4 单人可玩与动态难度（核心体验保障）

**设计目标：任何人数都能立刻开一把且爽感不打折。** 等不到好友、只想自己摸一把鱼时，也能秒进、秒爽。

**PvP — 人机补位（Bot Fill）**

- 真人不足开局人数时，自动用 AI 机器人填满两队，保证对枪密度。
- 机器人难度档位：**萌新 / 普通 / 老兵**（影响反应时间、瞄准精度、走位），房主可调。
- 真人随时可加入（房间码/匹配），新真人进来时机器人**自动腾位**，体验无缝。
- 机器人行为：巡逻、索敌、寻掩体、对枪、偶尔失误（拟人化，不做"神枪机器人"破坏爽感）。

**PvE — 按真人数量动态缩放（人越多，怪越多越强）**

刷怪量与强度随**当前真人数量（1–8）**实时缩放，1 人轻松割草，8 人则是高强度守关：

| 缩放维度 | 随人数变化 |
|---|---|
| 每波丧尸总量 | 人数越多越多（系数表覆盖 1–8 人，如 1 人 ×0.6 起、8 人 ×3.0+，受同屏上限约束分波涌入） |
| 丧尸血量/伤害 | 随人数小幅上调，避免人多碾压 |
| 精英/特感占比 | 人数越多，高级怪比例越高 |
| BOSS 血量与技能强度 | 按人数分档放大（1–8 各一档） |
| 金币掉落 | 同步微调，保证人均成长节奏一致 |

- **中途进出动态调整**：玩家中途加入/掉线时，下一波刷怪量平滑过渡，不会突然过载或空场。
- **单人特殊照顾**：1 人时同屏怪量、近身围殴密度下调，给足走位空间；保留 §2.2 的单人限次自救。
- 所有缩放系数进 `shared/config`，可热调（详见技术文档波次生成器）。

> 一句话：**1 个人进去是"爽快割草"，8 个人进去是"刺激守关"**，两端体验都成立。

## 3. 局内节奏拆解（以 PvE 8 分钟为例）

| 时间 | 阶段 | 设计意图 |
|---|---|---|
| 0:00–0:20 | 进场、武器到手 | 零等待，立刻能开枪 |
| 0:20–2:00 | 波次 1–2（杂兵） | 建立手感与爽感，掉第一批金币 |
| 2:00–2:30 | 间歇 1：升级 | 第一次"变强"，正反馈 |
| 2:30–5:30 | 波次 3–4（精英/特感） | 拉升强度，逼迫用成长应对 |
| 5:30–6:00 | 间歇 2：升级 | 第二次质变 |
| 6:00–8:00 | BOSS 波 | 高潮，团队配合，爆装/结算 |

PvP 则用"短复活 + 连杀奖励 + 限时"维持每 10–20 秒一个小高潮。

## 4. 爽点系统（核心，重点投入）

### 4.1 爆头特效（爽点 ①）

命中**头部碰撞体**时触发组合反馈：

- **视觉**：屏幕中央红色 `HEADSHOT` 飘字 + 骷髅图标，命中点爆血粒子，准星瞬间放大/红闪。
- **顿帧/慢镜**：被爆头者死亡瞬间 80–120ms 轻微顿帧 + 击杀者镜头微缩放（zoom punch）。
- **音效**：清脆"叮/咔嚓"爆头专属音 + 低频"咚"确认击杀（CF 风格、原创/CC 素材）。
- **击杀图标**：击杀提示条上用"爆头"专属图标（区别于普通击杀）。

### 4.2 音效系统（爽点 ②）

| 事件 | 设计 |
|---|---|
| 开火 | 每把枪独立音色，近/远/室内混响差异 |
| 上膛/换弹 | 机械感分层（拉栓、弹匣落地、推弹到位三段） |
| 命中反馈 | 命中敌人有"打肉/打甲"确认音 + 准星 hitmark |
| 中弹（自己） | 方向性受击音 + 屏幕血边 + 轻震屏 |
| 连杀播报 | 双杀/三杀/暴走/无敌（CF 式语音播报，原创配音或 CC 素材） |
| 环境 | 丧尸低吼、远处枪声、BOSS 咆哮 |

> 全部音效走 **WebAudio + 空间化（PannerNode）**，做到"听声辨位"。

### 4.3 受击与击杀反馈

- **HitMarker**：命中即在准星显示 X，爆头显示红色 X。
- **击杀回收**：击杀瞬间敌人轮廓高亮一帧 + 金币/积分数字弹出（PvE）。
- **连杀计数**：右侧连杀条，连杀越高边框越燃。

## 5. 武器系统

### 5.1 首期武器表（4–6 把，覆盖主流手感）

| 武器 | 类型 | 定位 | 特点 |
|---|---|---|---|
| 突击步枪 AR | 全自动 | **默认主武器** | 均衡，易上手 |
| 冲锋枪 SMG | 全自动 | 近身游走 | 射速高、移速快、远距衰减 |
| 霰弹枪 SG | 泵动 | PvE 割草 | 近距爆发、刷尸爽 |
| 狙击枪 SR | 栓动 | 爆头流 | 一枪爆头、开镜慢 |
| 手枪 Pistol | 半自动 | 副武器/兜底 | 换枪快、无限补给 |
| （PvE 限定）机枪 LMG | 全自动 | 守点压制 | 大弹匣、压制 BOSS |

### 5.2 武器数值维度

伤害（头/胸/肢倍率）、射速、弹匣容量、备弹、换弹时间、后坐力曲线、扩散、有效射程、移速修正。

### 5.3 升级树（局外永久 + PvE 局内临时）

每把枪 3 条线：**伤害**、**操控**（后坐/换弹/移速）、**容量**（弹匣/备弹）。
PvE 局内用金币即时升级（本局有效）；局外用永久货币解锁基础属性与皮肤。

## 6. 丧尸设计（爽点 ③ 的载体）

### 6.1 分级表

| 等级 | 名称 | 行为 | 血量 | 奖励 | 出现 |
|---|---|---|---|---|---|
| T1 | 普通尸 | 直线慢走、近身抓 | 低 | 金币 × | 全程 |
| T2 | 奔尸 | 快速冲刺 | 中低 | 金币 ×× | 波 2+ |
| T3 | 精英·肥尸 | 高血、近身重击、死亡溅射 | 高 | 金币 ××× | 波 3+ |
| T4 | 特感·吐酸 | 远程吐酸、减速场 | 中 | 金币 ××× + 概率掉道具 | 波 4+ |
| T5 | BOSS·尸王 | 多阶段、范围技、召唤小怪 | 极高 | 大量金币 + 结算装备/皮肤碎片 | BOSS 波 |

### 6.2 爽感要点

- **可见弱点**：头部/弱点高亮，鼓励精准 → 爆头爽点联动。
- **成群密度**：杂兵成波涌来制造"割草感"，但服务器要控制实体数（性能预算见技术文档）。
- **掉落即拾**：金币自动吸附拾取，不打断节奏。
- **死亡表现**：分级不同的死亡动画/破碎/爆体（性能友好的粒子方案）。

## 7. 成长与经济系统

### 7.1 双层经济

| | 局内（PvE） | 局外（永久） |
|---|---|---|
| 货币 | 金币（本局清零） | 积分/钻石（持久） |
| 来源 | 击杀丧尸、过波奖励 | 局末结算、每日首胜、任务 |
| 用途 | 即时升枪、买技能、补弹 | 解锁武器、永久属性、皮肤、表情 |
| 设计意图 | 局内正反馈飞轮 | 留存与回访 |

### 7.2 技能系统（PvE 局内可买）

- **生存**：移速+ / 回血 / 倒地自救。
- **输出**：暴击率+ / 换弹速度+ / 爆头伤害+。
- **团队**：范围治疗 / 弹药共享 / 嘲讽光环。

> 技能用"局内金币即时购买、本局生效"，让每局都有 build 变化，避免无脑突突。

### 7.3 防数值膨胀

PvP **不带永久数值优势**（公平竞技），永久成长只给 PvP **外观/便利**，避免"充钱变强"破坏对枪公平。数值养成集中在 PvE。

## 8. 操作方案

### 8.1 PC（键鼠）

- 移动 WASD、跳跃 Space、蹲 Ctrl、奔跑 Shift。
- 鼠标视角、左键开火、右键开镜、R 换弹、Q 换枪、F 互动/扶人、数字键选武器。
- Pointer Lock API 锁定鼠标。

### 8.2 移动端（虚拟手柄 + 按钮）

- **左摇杆**：移动；**右屏滑动**：转视角。
- 右下按钮组：开火（大）、开镜、换弹、跳、蹲、互动。
- 可选**陀螺仪微调**瞄准、**自动开火**（瞄准敌人时辅助射击，降低门槛）。
- 自适应布局，按钮可自定义位置；按设备性能自动降画质。

### 8.3 设备识别

进入即检测触屏/指针，自动切换 UI 与控制方案；支持手柄（Gamepad API）作为加分项。

## 9. 地图设计

### 9.1 首期地图

| 地图 | 模式 | 风格 | 设计要点 |
|---|---|---|---|
| 仓库（Warehouse） | PvP-TDM | 中近距、对称 | 三线对枪、集装箱掩体、爆头视线点 |
| 废街（Dead Street） | PvE | 直线推进 | 血战上海滩式线性关卡 + 守点房 |

### 9.2 原则

- **小而紧凑**：跑图 ≤ 15 秒能遇敌，杜绝空旷找人。
- **对称公平**（PvP）：双方资源/视线对等。
- **引导清晰**（PvE）：明确推进方向 + 守点高亮。
- **掩体节奏**：掩体密度服务"探头对枪"的爽感。

## 10. UI / UX 与社交

### 10.1 开局流程（≤30 秒）

```
落地页 → 输入昵称（或游客）→ [创建房间 / 输入房间码加入] →
房间内选模式/地图 → 房主开始 → 进入对局
```

- **邀请**：房间生成短链/口令，复制发好友，点开即进同房。
- **一键单人开局**：落地页提供"单人快速开打"，跳过等人，PvP 自动人机补位、PvE 自动按 1 人难度起（见 §2.4）。
- **快速填充**：人不够时 PvP 自动人机补位、PvE 直接开；真人随时加入无缝替换机器人 / 实时提升难度。

### 10.2 HUD

血量/护甲、弹药、小地图（可选）、连杀条、击杀信息流、计分板（Tab）、金币（PvE）。

### 10.3 局末结算

战绩（击杀/爆头/伤害）、MVP、金币/经验、升级解锁、再来一局/换模式按钮（一键重开，强化"再来亿把"）。
