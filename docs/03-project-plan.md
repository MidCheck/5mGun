# 03 · 项目计划与里程碑

> 版本 v0.1 · 状态：待评审
> 策略：**先纵切打通一条最小可玩链路（单机射击 → 多人同步 → 爽点 → PvE 成长），再横向铺量。** 每个里程碑都产出"能真实玩到"的东西。

---

## 里程碑总览

| 里程碑 | 目标 | 产出（可验收） |
|---|---|---|
| **M0** 脚手架 | 工程可运行 | 前后端项目跑通、共享配置、CI、空场景能进 |
| **M1** 单机射击手感 | 一个人能爽快开枪 | 第一人称移动+开火+换弹+命中假人+音效+爆头特效（无网络） |
| **M2** 多人同步 | 多人能同场对枪 | 权威服务器 + 预测/校正/插值 + PvP-TDM 基础对局 |
| **M3** 爽点与对局闭环 | PvP 一局完整可玩 | 滞后补偿命中、连杀播报、计分、复活、结算、房间/邀请 |
| **M4** PvE 与成长 | 刷尸+变强闭环 | 丧尸分级AI、波次、金币、局内升级/技能、结算永久成长 |
| **M5** 移动端与上线 | 触屏可玩、可部署 | 虚拟摇杆、性能档位、防作弊加固、云部署、压测 |

> 顺序刻意把"手感"和"网络同步"这两个最高风险项前置（M1、M2），尽早暴露问题。

---

## M0 · 工程脚手架

**目标**：把技术栈搭起来，能 `npm run dev` 同时起前后端并连上。

- [ ] Monorepo（pnpm workspaces）：`client/` `server/` `shared/`
- [ ] `client`：Vite + TS + Three.js，渲染一个空场景 + 相机
- [ ] `server`：Node + Colyseus，一个空 `TestRoom`
- [ ] `shared`：数值/武器/丧尸配置 JSON + 类型，前后端共用
- [ ] 客户端能连上服务器房间、收发心跳
- [ ] 代码规范（eslint/prettier）、基础 CI、README 跑通说明

**验收**：浏览器打开能进场景，控制台显示已加入服务器房间。

---

## M1 · 单机射击手感（最高优先，决定"爽不爽"）

**目标**：纯客户端（先不联网）做出"开枪很爽"的核心手感。

- [ ] 第一人称控制器：WASD 移动、鼠标视角（PointerLock）、跳/蹲/奔跑、胶囊体碰撞
- [ ] 武器视图模型：持枪、开火动画、后坐、枪口火光、换弹动画
- [ ] 射击：hitscan 射线、tracer 表现、扩散/后坐曲线
- [ ] 训练假人：带头/躯干/四肢分段碰撞体，命中倍率
- [ ] **爆头特效**：HEADSHOT 飘字 + 粒子 + 顿帧/zoom punch
- [ ] **音效**：开火/换弹/命中/爆头/中弹（占位 CC 素材）+ WebAudio 空间化
- [ ] HitMarker、伤害飘字、准星反馈
- [ ] 1 把默认 AR + 手枪

**验收**：一个人在场景里打假人，爆头有完整视听爽点，手感跟手。

---

## M2 · 多人同步（最高风险，决定"能不能联机"）

**目标**：把 M1 的手感搬到权威服务器架构下，多人同场不卡不飘。

- [ ] 服务器移动模拟（权威）+ 玩家 schema 同步
- [ ] **客户端预测 + 服务器校正（回滚重放）**
- [ ] **远端实体插值**（~100ms 延迟平滑）
- [ ] 输入 seq 协议、快照下行、增量同步
- [ ] 多人同房可见、可移动、能互相看到开枪（先打通 ~8 人，再压测扩到 PvP 上限 20 人）
- [ ] 网络抖动模拟测试（注入 50–150ms + 丢包）

**验收**：两人不同网络下同场移动平滑、开枪表现一致；本地手感不被网络拖累。

---

## M3 · 爽点与 PvP 对局闭环

**目标**：PvP-TDM 一局从进房到结算完整可玩。

- [ ] **滞后补偿命中**（历史回溯射线判定）+ 服务器伤害结算
- [ ] 队伍/出生点/出生保护、3 秒复活
- [ ] 计分（限时 6min / 40 杀）、击杀信息流、计分板
- [ ] **连杀播报**（双杀/三杀…）、连杀条
- [ ] 房间系统：创建/邀请码加入
- [ ] **PvP 人机补位（Bot Fill）**：服务器侧机器人 AI（巡逻/索敌/寻掩体/对枪/换弹）、难度档位（萌新/普通/老兵）、真人进出自动增删与顶替 → **保证 1 人也能立刻开打**
- [ ] 局末结算页（战绩/MVP/再来一局）
- [ ] 仓库地图（PvP 对称图）

**验收**：**1 个人就能用人机补位开打**；多人（目标上限 20）能完整打完一局 TDM，爆头/连杀/结算爽点齐全，20 人房同步与 tick 在预算内。

---

## M4 · PvE 丧尸与成长经济

**目标**：组队刷尸 + 即时变强的飞轮跑通。

- [ ] 丧尸 AI：索敌/寻路（流场）/攻击，T1–T4 分级 + BOSS
- [ ] 波次生成器（5 波 + BOSS）、难度曲线、波间歇
- [ ] **按真人数动态难度缩放**：怪量/血量/伤害/精英占比/BOSS/金币系数表（`shared/config` 热调）、中途进出下一波平滑过渡
- [ ] 金币掉落/自动拾取、局内升级商店、技能购买（本局生效）
- [ ] 倒地/扶起协作机制 + **单人限次自救**
- [ ] 局末结算 → 永久经验/货币/解锁写库
- [ ] 废街地图（线性推进 + 守点）
- [ ] InstancedMesh 渲染丧尸群 + 实体上限/AOI（单人下调同屏密度）

**验收**：**1 人能轻松割草、8 人需要配合守关**，难度随人数（1–8）平滑缩放；金币→升级→变强闭环成立，BOSS 波有高潮。

---

## M5 · 移动端、加固与上线

**目标**：触屏可玩、性能达标、可部署上线。

- [ ] 虚拟摇杆 + 按钮 UI、设备自适应、可选 aim assist
- [ ] 画质档位（高/中/低）、低端机优化、移动端 30+ FPS
- [ ] 防作弊加固（位移/射速/命中率监控与钳制、限流）
- [ ] 持久化迁移 SQLite→PostgreSQL、（按需）Redis 匹配
- [ ] 云部署（Nginx/WSS/CDN）、监控告警、压测（并发房间）
- [ ] 落地页、新手引导、敏感词过滤

**验收**：手机浏览器流畅可玩，线上环境多人对局稳定，基础风控到位。

---

## 横向工作流（贯穿各阶段）

- **数值调优**：每个里程碑回看手感/节奏/时长，调 `shared/config`。
- **素材替换**：占位 → 正式 CC/原创素材（见 [04](04-assets.md)），上线前清干净版权。
- **测试**：每局时长是否落在 5–10min？爽点反馈是否到位？早邀请真人小范围试玩。

---

## 建议的最省钱起步路径

1. 本地 / 1 台便宜云主机跑 M0–M4，验证玩法（不急着弹性扩容）。
2. 玩法被验证"好玩"后，再投入 M5 的部署与扩展成本。
3. 单一权威服务器进程即可支撑早期数百并发，**先验证、后扩容**。
